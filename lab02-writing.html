Lab 2 Reflection

In lab 2, I tested two AI tools, ChatGPT and Perplexity, to see what happens when you push them outside the kinds of prompts they usually get. I wasn’t trying to get the best answer out of these tools, but I was trying to see how they behave when the prompt is confusing, contradicting, or somewhat impossible. I gave both tools the same set of prompts so I could compare them directly, and I took screenshots of the outcomes and responses I was given.

The first experiment was about assumptions. I asked each tool to ask me six clarifying questions before it was allowed to help. Both tools did it pretty easily, which showed me that if you give clear instructions, they can follow a structure. The most interesting part of this experiment was how fast they started making guesses about me and what I wanted. ChatGPT leaned toward a more analytical, evidence-based tone and acted like I already had a portfolio plan in place. Perplexity asked helpful questions, too, but sounded more like a writing coach and focused on how the final result would come across to a reader. Even with the same prompt, both tools quickly started shaping the task into what they thought success should look like and what they thought I would be happy with, which reminded me that AI frames the situation as it responds.

The second experiment was meant to find the breaking point or the moment where a tool should either admit it can’t do something or risk making something up. I gave both tools a prompt with rules that didn’t work together by asking for direct quotes and APA citations from credible sources, but also said they weren’t allowed to use or access any sources. ChatGPT basically said that it was impossible and treated making up quotes as an ethical issue. It gave a best-effort response while clearly saying what it couldn’t do. On the other hand, Perplexity didn’t address the contradiction. It generated quotes and a reference list anyway, which looked real at first glance, but there was no way to tell if any of it really was. That was the biggest takeaway for me was that one tool showed its limits, and the other created something that looked credible even when it shouldn’t have been able to, so the user was pleased.

The third experiment was more creative. I asked each tool to redesign its own interface in a way that would stop people from trusting it too much. ChatGPT suggested features like an assumptions panel, confidence labels, and a use with care summary so users can see uncertainty upfront. Perplexity focused more on verification tools, like verify this buttons, an evidence mix panel, and a citation quality meter. What stood out to me is that almost every feature that reduces overtrust also adds extra clicks, extra reading, or extra steps. This seems like a tradeoff for users when they have to give up the speed and convenience for responsible use.

Overall, Lab 2 showed me that AI tools are built to feel helpful and convenient, not to guarantee truth. The biggest difference between these two was how they handle uncertainty and constraints. ChatGPT was more likely to admit that a prompt didn’t make sense and avoid making things up. Perplexity was more likely to give a polished output that looked like research, even when it wasn’t verifiable, to please the user. I also learned that interface design is important because it encourages speed and makes it easy to accept a confident answer without stopping to check it.

Going forward, I want to use AI tools as helpers that support my thinking, instead of shortcuts that replace it. I can use these tools to brainstorm research questions, turn project goals into measurable steps, draft plans for collecting or organizing data, and generate multiple interpretations of the same pattern so I can test them. I can also use them to help debug code, write clearer comments, and make my workflow easier to understand. But I don’t want to treat AI as an authority for facts, quotes, or citations. I learned from this that sources need to be checked. When I try new tools, I’m going to pay attention to whether they make verification easy, how they show uncertainty, what the interface encourages me to skip, and how they handle bias or perspective.
